# Feature: README & API Documentation

## Overview

Update the project README with challenge-specific documentation: project description, architecture overview, prerequisites, build/run/test commands, API endpoint reference with example curl commands, and design decisions. Swagger/OpenAPI docs are auto-generated by FastAPI via Pydantic models and require no additional work.

## User Story

As a judge, I want to read clear setup and usage instructions so that I can build, run, and test the solution independently.

## Rationale

A clear README is a submission requirement and a production-readiness signal. Judges will read it before looking at code. Good documentation demonstrates engineering maturity and makes it easy for evaluators to verify the solution works. Swagger docs come free from FastAPI and show all request/response models interactively.

## Acceptance Criteria

- [ ] README includes project description (what PennyWise does, the problem it solves)
- [ ] README includes architecture overview (routes -> services -> models, stateless design)
- [ ] README includes prerequisites (Docker)
- [ ] README includes build command: `docker build -t blk-hacking-ind-shamb-<lastname> .`
- [ ] README includes run command: `docker run -d -p 5477:5477 blk-hacking-ind-shamb-<lastname>`
- [ ] README includes test command: `pytest test/ -v`
- [ ] README includes API endpoint reference for all 5 endpoints with example curl commands
- [ ] README includes design decisions (why FastAPI, why stateless, precision handling)
- [ ] Swagger docs are auto-generated and accessible at `/docs`
- [ ] All request/response models appear in Swagger with field descriptions

## Implementation Details

### Approach

Update the existing `README.md` at the project root with a challenge-focused structure. Keep it concise but complete. Include curl examples for each endpoint so judges can copy-paste and test immediately. The Swagger docs are already provided by FastAPI -- just ensure all Pydantic models have proper field descriptions.

### LLM/Processing Configuration

**Type:** Deterministic (No LLM)

**Processing Type:**

- Verify Swagger docs render correctly at `/docs`

### Components Affected

- Root: `README.md` (modified -- add challenge sections)
- Backend: `backend/src/models/challenge.py` (verify Pydantic models have `description` fields for Swagger)

### API Changes

No new endpoints. Documentation of existing endpoints.

### Database Changes

None. Stateless computation.
